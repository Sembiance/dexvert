Zoetrope Animation RIFF/VRUN file format
=========================================

Scope
-----

This describes the Zoetrope Animation files in sample0/.  The files use the
ASCII magic "RIFF", but the Zoetrope container fields are big-endian.  The
stored picture data is Amiga-style bitplane video with a per-frame 12-bit RGB
palette.  No sample contains audio or any audio-related chunk; the converter
therefore emits silent AVI files.

One sample, sample0/21003/supermanani.rif, is physically truncated: its first
VRUN chunk declares 0x30e4 payload bytes at file offset 0x88, but the file ends
before that payload is complete.  The converter treats this specific class of
EOF truncation leniently after the VRUN header has been validated: it decodes
all complete byte-vertical operations available, leaves missing mode-1 data as
cleared pixels, leaves missing mode-2 data as previous-frame pixels, emits the
salvaged frame, and stops at EOF.  Other malformed tags, dimensions, reserved
fields, unsupported compression modes, and non-EOF structural errors still fail.
sample0/11573/sachiko.riff contains 34 valid declared frames and then an
unreferenced partial extra VRUN fragment; the declared animation is converted,
and the trailing fragment is documented below as damaged residue outside the
declared frame set.

Top-level layout
----------------

All multi-byte fields in this section are big-endian.

Offset  Size  Meaning
0x00    4     ASCII "RIFF".
0x04    4     Declared RIFF payload size.  For most files this is file_size-8,
              but several samples contain stale or incorrect values.  The frame
              table and VRUN chunk sizes are authoritative for decoding.
0x08    4     Reserved, always 0 in handled samples.
0x0c    2     Width in pixels.  All samples are 320.
0x0e    2     Height in pixels.  All samples are 200.
0x10    2     Bitplane depth.  Samples are 5 planes except ssachiko2.riff,
              which is 4 planes.
0x12    2     Reserved, always 0.
0x14    2     Primary frame count.
0x16    2     Frame delay in Amiga jiffies, 1/60 second units.  A zero value
              means no explicit delay was stored; the converter uses Zoetrope's
              observed default of 4 jiffies (15 fps) for AVI timing.
0x18    2     Secondary frame count.  It is either 0 or equal to the primary
              frame count in every sample.  Decoding uses the primary count.
0x1a    38    Reserved, always 0.
0x40    4*n   Frame offset table, n = primary frame count.  Entries are
              absolute byte offsets from the beginning of the file to VRUN
              chunks.

Normal files have a populated offset table.  sachiko.riff has an all-zero
table; its 34 declared VRUN chunks are stored sequentially starting immediately
after the table at 0x40 + 4*n.  That is a structural variant in the sample, not
a scan heuristic: the declared count and each chunk's declared size determine
the 34 frame boundaries exactly.

VRUN chunk layout
-----------------

All multi-byte fields in this section are big-endian.  A VRUN chunk starts at a
frame offset from the table or sequential derivation:

Offset from chunk  Size  Meaning
0x00               4     ASCII "VRUN".
0x04               4     Payload size in bytes.
0x08               ...   Payload, exactly the declared size.

VRUN payload layout:

Payload offset  Size  Meaning
0x00            4     Reserved, always 0.
0x04            2     Width; matches top-level width.
0x06            2     Height; matches top-level height.
0x08            2     Bitplane depth; matches top-level depth.
0x0a            2     Reserved, always 0.
0x0c            20    Five plane descriptors.  Each descriptor is:
                      u16 compression_mode, u16 byte_count.
                      Descriptors beyond the stored depth are (0, 0).
0x20            16    Reserved, always 0.
0x30            64    Palette: 32 big-endian 12-bit Amiga RGB words.
                      Bits 11..8 are red, 7..4 green, 3..0 blue.
                      Expand each nibble to 8-bit color by multiplying by 17.
0x70            ...   Plane byte streams in descriptor order.  The byte_count
                      values account for every stream byte.

The payload size is exactly:

    0x70 + sum(all five descriptor byte_count values)

For a 4-plane file the fifth descriptor byte_count is zero.

Plane data and alignment bytes
------------------------------

Each plane is 320/8 = 40 byte-columns by 200 rows.  Compression is vertical:
the stream describes column 0 from top to bottom, then column 1, through column
39.  Each column starts with an 8-bit op count.  Descriptor byte_count values
include the compressed column data plus 0 to 3 alignment/trailer bytes.  The
decoder stops after all 40 columns; the remaining 0 to 3 bytes are counted
alignment bytes.  In the samples those alignment bytes are often zero but are
not reliably initialized, so their content has no image meaning.

For an otherwise valid file that ends inside the final VRUN payload, the
converter's salvage path decodes only complete operations.  A truncated
operation stops the current plane, and absent later planes are left in their
initial state for that compression mode.  This behavior is used by the damaged
supermanani.rif sample.

Compression mode 1: full-frame vertical RLE
-------------------------------------------

Mode 1 reconstructs the entire plane from scratch into a cleared destination.
For each of 40 columns:

1. Read op_count.
2. Repeat op_count operations:
   - If op byte has bit 7 set, count = op & 0x7f.  Copy the next count bytes
     literally into successive rows of this column.
   - If op byte has bit 7 clear, count = op.  Read one value byte and repeat it
     count times into successive rows of this column.
3. The operations must fill exactly 200 rows.

The first frame in every valid sample uses mode 1 for each active plane.

Compression mode 2: byte-vertical delta
---------------------------------------

Mode 2 is Jim Kent's byte-vertical delta coding as used by Amiga ANIM method 5.
It applies changes to the same plane from the previous decoded frame.  For each
of 40 columns:

1. Read op_count.  If it is 0, the column is unchanged.
2. Repeat op_count operations with a current row cursor:
   - Skip op: op is 1..127.  Advance the row cursor by op rows, leaving those
     bytes unchanged.
   - Literal op: op has bit 7 set.  count = op & 0x7f.  Copy the next count
     bytes into successive rows of this column.
   - Same op: op is 0.  Read count byte and value byte.  Store value into the
     next count successive rows of this column.
3. The row cursor may stop before row 200; the remaining rows are unchanged.

Image conversion
----------------

The decoded planes are Amiga planar bitmap data.  For a pixel at x,y:

1. byte_index = y * (width/8) + floor(x/8)
2. bit = 7 - (x % 8)
3. For each plane p, bit p of the chunky palette index is:
       (plane[p][byte_index] >> bit) & 1
4. Look up the resulting index in the frame's 32-entry palette.

The AVI converter writes 24-bit uncompressed DIB frames.  No audio is emitted
because the Zoetrope samples contain no audio bytes or audio chunk type.
