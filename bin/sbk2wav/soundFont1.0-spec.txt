SoundFont 1.0 (.SBK) File Format Specification
================================================

Version: 1.0
Target Hardware: E-mu Systems EMU8000 synthesizer chip (Creative Labs AWE32/AWE64)
File Extension: .SBK
Byte Order: Little-endian throughout


1. OVERVIEW
-----------

A SoundFont 1.0 file is a RIFF (Resource Interchange File Format) container with
the form type "sfbk". It contains three mandatory LIST chunks:

  RIFF('sfbk')
  +-- LIST('INFO')    Metadata about the bank
  +-- LIST('sdta')    Sample data
  +-- LIST('pdta')    Preset, instrument, and sample header data

The file stores wavetable synthesis data for the EMU8000 chip: digitized audio
samples along with the preset/instrument/zone hierarchy that controls how those
samples are played in response to MIDI events.

Unlike SoundFont 2.0 (.SF2), the SBK format:
  - Uses 16-byte sample headers (no sample rate, pitch, or link fields in shdr)
  - Stores sample names in a separate 'snam' chunk (not inside shdr)
  - Uses generator 55 to store sample playback rate (reserved/unused in SF2)
  - Uses 6-byte modulator records (vs 10 bytes in SF2)
  - Does NOT require word-alignment padding between chunks


2. RIFF CONTAINER
-----------------

Offset  Size  Field       Description
------  ----  -----       -----------
0       4     ckID        "RIFF" (0x52 0x49 0x46 0x46)
4       4     ckSize      Size of remaining data after this field (fileSize - 8)
8       4     formType    "sfbk" (0x73 0x66 0x62 0x6B)

The three LIST chunks follow immediately. There is NO word-alignment padding
between chunks in SBK files (unlike standard RIFF which pads odd-sized chunks
to even boundaries). Some SBK creation tools may include padding; parsers
should be tolerant of both.


3. LIST 'INFO' - Metadata
--------------------------

The INFO list contains sub-chunks describing the bank. All string values are
null-terminated ASCII (ZSTR). Sub-chunks appear in the order listed, though
not all are required.

Sub-chunk  Size     Required  Description
---------  ----     --------  -----------
ifil       4        Yes       File version. Two WORDs: major, minor.
                              Always 0x0001, 0x0000 for SoundFont 1.0.
isng       varies   Yes       Target sound engine name (ZSTR).
                              Typically "EMU8000\0" (8 bytes).
irom       varies   No        ROM sample set name (ZSTR).
                              Typically "1MGM\0" or "1MGM\0\0" (5-6 bytes).
iver       4        No        ROM version. Two WORDs: major, minor.
                              Typically 2.8 (0x0002, 0x0008).
INAM       varies   No        Bank name (ZSTR). e.g., "General MIDI\0".
IPRD       varies   No        Target product (ZSTR). Typically "SBAWE32\0".
IENG       varies   No        Sound designer/engineer name (ZSTR).
ISFT       varies   No        Creation software (ZSTR).
                              e.g., "SFSTORE.DLL v1.0\0".
ICRD       varies   No        Creation date (ZSTR). Free-form text.
ICOP       varies   No        Copyright string (ZSTR).

Chunk format (each sub-chunk within INFO):

  Offset  Size  Field     Description
  ------  ----  -----     -----------
  0       4     ckID      Four ASCII characters (e.g., "ifil")
  4       4     ckSize    Size of data following this field
  8       N     ckData    Chunk data (N = ckSize bytes)


4. LIST 'sdta' - Sample Data
------------------------------

The sdta list contains two sub-chunks:

4.1 'snam' - Sample Names
~~~~~~~~~~~~~~~~~~~~~~~~~~

An array of fixed-size 20-byte records, one per sample in the bank.

  Offset  Size  Field       Description
  ------  ----  -----       -----------
  0       20    szName      Null-terminated ASCII sample name, padded with
                            zeros to exactly 20 bytes.

The number of samples is: snam_chunk_size / 20.
Sample indices are 0-based and correspond 1:1 with shdr records in pdta.

Sample names may contain any printable ASCII characters. Names like
"Sample 0:0:0" are auto-generated by some tools; others use descriptive
names like "kpianob1" or "FINGBAS1".

4.2 'smpl' - Sample Data
~~~~~~~~~~~~~~~~~~~~~~~~~~

Raw audio sample data. Format:

  - 16-bit signed integers, little-endian
  - Mono (single channel)
  - Contiguous: all samples are stored sequentially

Samples are arranged sequentially with inter-sample padding gaps (typically
40-76 zero samples) between them. These gaps are required by the EMU8000
hardware for its interpolation engine. The exact gap size may vary.

Sample offsets in the shdr records (see section 5.8) are in units of
sample frames (not bytes). To convert to a byte offset within the smpl
chunk: byte_offset = sample_frame_offset * 2.

If the smpl chunk is absent, all samples reference the EMU8000's built-in
ROM sample set and no user audio data is embedded in the file.


5. LIST 'pdta' - Preset Data
------------------------------

The pdta list contains nine sub-chunks that define the synthesis hierarchy.
They MUST appear in the following order:

  phdr -> pbag -> pmod -> pgen -> inst -> ibag -> imod -> igen -> shdr

The hierarchy is:

  Presets (phdr)
    -> Preset Zones/Bags (pbag)
      -> Preset Modulators (pmod)
      -> Preset Generators (pgen)
        -> Instruments (inst)
          -> Instrument Zones/Bags (ibag)
            -> Instrument Modulators (imod)
            -> Instrument Generators (igen)
              -> Sample Headers (shdr)
                -> Sample Data (smpl)
                -> Sample Names (snam)

5.1 'phdr' - Preset Headers
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 38-byte records. The last record is always a terminal entry with
name "EOP" (End Of Presets).

  Offset  Size  Field          Description
  ------  ----  -----          -----------
  0       20    achPresetName  Null-terminated ASCII preset name, zero-padded
                               to 20 bytes. e.g., "Piano 1", "Inst 0".
  20      2     wPreset        MIDI preset number (0-127 for standard GM).
  22      2     wBank          MIDI bank number (usually 0).
  24      2     wPresetBagNdx  Index into pbag of this preset's first zone.
  26      4     dwLibrary      Reserved/unused in most implementations.
  30      4     dwGenre        Reserved/unused in most implementations.
  34      4     dwMorphology   Reserved/unused in most implementations.

Number of presets: phdr_chunk_size / 38 (includes terminal "EOP" entry).

A preset's zones span from pbag[wPresetBagNdx] to
pbag[next_preset.wPresetBagNdx - 1].

5.2 'pbag' - Preset Bags (Zones)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 4-byte records. Each record defines one preset zone.

  Offset  Size  Field        Description
  ------  ----  -----        -----------
  0       2     wGenNdx      Index into pgen of this zone's first generator.
  2       2     wModNdx      Index into pmod of this zone's first modulator.

A zone's generators span from pgen[wGenNdx] to
pgen[next_zone.wGenNdx - 1].

5.3 'pmod' - Preset Modulators
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 6-byte records. Each record defines one modulator routing.

  Offset  Size  Field          Description
  ------  ----  -----          -----------
  0       2     sfModSrcOper   Source modulation operator/controller.
  2       2     sfModDestOper  Destination generator to modulate.
  4       2     modAmount      Signed modulation amount.

The terminal record is all zeros (6 bytes of 0x00). In practice, SBK files
rarely contain active preset modulators; most files have only the terminal
record (pmod size = 6).

5.4 'pgen' - Preset Generators
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 4-byte records. Each record sets one synthesis parameter for a
preset zone.

  Offset  Size  Field       Description
  ------  ----  -----       -----------
  0       2     sfGenOper   Generator operator (see Section 6).
  2       2     genAmount   Value (interpretation depends on operator).

The genAmount field is interpreted as:
  - Signed 16-bit integer (INT16) for most generators
  - Two unsigned bytes (hi, lo) for range generators (keyRange, velRange)

Key preset-level generators:
  - 41 (instrument): Index into inst array. Links this zone to an instrument.
  - 43 (keyRange): MIDI key range (lo=low key, hi=high key).
  - 55 (sampleRate): Default playback rate in Hz (SF1-specific, can be
    overridden at instrument level).

The last generator in the entire pgen array is a terminal record (op=0, val=0).

5.5 'inst' - Instrument Headers
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 22-byte records. The last record is always a terminal entry with
name "EOI" (End Of Instruments).

  Offset  Size  Field          Description
  ------  ----  -----          -----------
  0       20    achInstName    Null-terminated ASCII instrument name,
                               zero-padded to 20 bytes.
  20      2     wInstBagNdx   Index into ibag of this instrument's first zone.

Number of instruments: inst_chunk_size / 22 (includes terminal "EOI" entry).

Note: Some SBK creation tools leave the name field uninitialized (garbage
data) for non-terminal records. The instrument name is informational only
and does not affect synthesis.

5.6 'ibag' - Instrument Bags (Zones)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 4-byte records. Each record defines one instrument zone.

  Offset  Size  Field        Description
  ------  ----  -----        -----------
  0       2     wGenNdx      Index into igen of this zone's first generator.
  2       2     wModNdx      Index into imod of this zone's first modulator.

Structure is identical to pbag. A zone's generators span from
igen[wGenNdx] to igen[next_zone.wGenNdx - 1].

5.7 'imod' - Instrument Modulators
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 6-byte records. Structure identical to pmod (see Section 5.3).
Most SBK files contain only the terminal record (imod size = 6).

5.8 'igen' - Instrument Generators
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 4-byte records. Each record sets one synthesis parameter for an
instrument zone. Structure identical to pgen (see Section 5.4).

Key instrument-level generators:
  - 53 (sampleID): Index into the shdr/snam arrays identifying which sample
    this zone uses. This is the terminal generator for each zone.
  - 55 (sampleRate): Playback sample rate in Hz. This SF1-specific generator
    defines the rate at which the sample data should be played back.
    Overrides any preset-level default.
  - 58 (overridingRootKey): MIDI key number at which the sample plays at its
    original pitch (0-127).

See Section 6 for the complete generator enumeration.

5.9 'shdr' - Sample Headers
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Array of 16-byte records. Each record describes one sample's location and
loop points within the smpl data. Records correspond 1:1 with snam entries
(sample N's name is in snam[N], its header is in shdr[N]).

  Offset  Size  Field          Description
  ------  ----  -----          -----------
  0       4     dwStart        Start offset in sample frames from the
                               beginning of the smpl chunk.
  4       4     dwEnd          End offset (exclusive) in sample frames.
                               The sample contains (dwEnd - dwStart) frames.
  8       4     dwStartloop    Loop start offset in sample frames (absolute,
                               from start of smpl chunk).
  12      4     dwEndloop      Loop end offset in sample frames (absolute).

All offsets are in sample frames (16-bit samples), not bytes.
Byte offset = frame_offset * 2.

The loop region is [dwStartloop, dwEndloop). When sample looping is enabled
(via generator 54, sampleModes = 1), the EMU8000 plays from dwStart to
dwEndloop, then jumps back to dwStartloop and repeats.

For non-looping samples, dwStartloop and dwEndloop may equal dwStart and
dwEnd respectively, or may point to a minimal loop region at the end of
the sample.

For ROM-only banks (no smpl chunk), the offsets refer to positions within
the EMU8000's built-in ROM sample memory.

Note: Unlike SoundFont 2.0, the shdr does NOT contain sample rate,
original pitch, pitch correction, sample link, or sample type fields.
These are handled through generators (igen/pgen) or are not applicable
in SF1.


6. GENERATOR ENUMERATION
--------------------------

Generators control synthesis parameters. Each is a 4-byte record:
2 bytes operator ID + 2 bytes value. Values are signed INT16 unless noted.

ID   Name                     Value Type    Description
--   ----                     ----------    -----------
 0   startAddrsOffset         INT16         Sample start address offset
                                            (in sample frames, added to shdr.dwStart).
 1   endAddrsOffset           INT16         Sample end address offset
                                            (added to shdr.dwEnd).
 2   startloopAddrsOffset     INT16         Loop start offset adjustment
                                            (added to shdr.dwStartloop).
 3   endloopAddrsOffset       INT16         Loop end offset adjustment
                                            (added to shdr.dwEndloop).
 4   startAddrsCoarseOffset   INT16         Sample start coarse offset
                                            (in units of 32768 samples).
 5   modLfoToPitch            INT16         Modulation LFO to pitch, in cents.
 6   vibLfoToPitch            INT16         Vibrato LFO to pitch, in cents.
 7   modEnvToPitch            INT16         Modulation envelope to pitch, in cents.
 8   initialFilterFc          INT16         Initial filter cutoff frequency.
                                            Units: 62.5 * 2^(val/128) Hz (approx).
 9   initialFilterQ           INT16         Initial filter resonance/Q factor.
10   modLfoToFilterFc         INT16         Mod LFO to filter cutoff, in cents.
11   modEnvToFilterFc         INT16         Mod envelope to filter cutoff, in cents.
12   endAddrsCoarseOffset     INT16         Sample end coarse offset
                                            (in units of 32768 samples).
13   modLfoToVolume           INT16         Mod LFO to volume, in centibels.
15   chorusEffectsSend        INT16         Chorus send amount (0-127).
16   reverbEffectsSend        INT16         Reverb send amount (0-127).
17   pan                      INT16         Stereo panning (0=hard left,
                                            64=center, 127=hard right).
21   delayModLFO              INT16         Modulation LFO delay, timecents.
22   freqModLFO               INT16         Modulation LFO frequency.
23   delayVibLFO              INT16         Vibrato LFO delay, timecents.
24   freqVibLFO               INT16         Vibrato LFO frequency.
25   delayModEnv              INT16         Modulation envelope delay, timecents.
26   attackModEnv             INT16         Mod envelope attack, timecents.
27   holdModEnv               INT16         Mod envelope hold, timecents.
28   decayModEnv              INT16         Mod envelope decay, timecents.
29   sustainModEnv            INT16         Mod envelope sustain level (0-127).
                                            0=full sustain, 127=instant decay.
30   releaseModEnv            INT16         Mod envelope release, timecents.
31   keynumToModEnvHold       INT16         Key-to-mod-env hold adjustment.
32   keynumToModEnvDecay      INT16         Key-to-mod-env decay adjustment.
33   delayVolEnv              INT16         Volume envelope delay, timecents.
34   attackVolEnv             INT16         Volume envelope attack, timecents.
35   holdVolEnv               INT16         Volume envelope hold, timecents.
36   decayVolEnv              INT16         Volume envelope decay, timecents.
37   sustainVolEnv            INT16         Volume envelope sustain (0-127).
                                            0=full sustain, 127=instant decay.
38   releaseVolEnv            INT16         Volume envelope release, timecents.
39   keynumToVolEnvHold       INT16         Key-to-vol-env hold adjustment.
40   keynumToVolEnvDecay      INT16         Key-to-vol-env decay adjustment.
41   instrument               UINT16        Index into inst array (pgen only).
43   keyRange                 RANGE         MIDI key range: lo byte = low key,
                                            hi byte = high key (0-127).
44   velRange                 RANGE         MIDI velocity range: lo = min,
                                            hi = max (0-127).
45   startloopAddrsCoarseOff  INT16         Loop start coarse offset
                                            (in units of 32768 samples).
46   keynum                   INT16         Fixed MIDI key number override.
47   velocity                 INT16         Fixed velocity override.
48   initialAttenuation       INT16         Initial volume attenuation (0-127).
                                            0=full volume, 127=silent.
50   endloopAddrsCoarseOffset INT16         Loop end coarse offset
                                            (in units of 32768 samples).
51   coarseTune               INT16         Coarse tuning in semitones.
52   fineTune                 INT16         Fine tuning in cents.
53   sampleID                 UINT16        Index into shdr/snam arrays (igen only).
                                            Identifies which sample this zone plays.
54   sampleModes              UINT16        Sample loop mode:
                                              0 = no loop
                                              1 = loop continuously
                                              3 = loop during key depression,
                                                  then play remainder
55   sampleRate               UINT16        Playback sample rate in Hz (SF1 only).
                                            Typical values: 5000-12000 Hz for
                                            EMU8000 wavetable data. This generator
                                            is reserved/unused in SoundFont 2.0.
56   scaleTuning              INT16         Scale tuning (cents/key, default 100).
57   exclusiveClass           INT16         Exclusive class for choke groups.
58   overridingRootKey        INT16         Root key override (MIDI note 0-127).
                                            The key at which the sample plays at
                                            its recorded pitch.


7. COMPLETE FILE LAYOUT (BYTE MAP)
------------------------------------

Every byte in an SBK file belongs to one of these structures:

  Byte Range                    Structure
  ----------                    ---------
  [0..3]                        RIFF chunk ID ("RIFF")
  [4..7]                        RIFF chunk size (DWORD, little-endian)
  [8..11]                       Form type ("sfbk")

  [12..15]                      LIST chunk ID ("LIST")
  [16..19]                      LIST chunk size
  [20..23]                      List type ("INFO")
  [24..24+INFO_size-1]          INFO sub-chunks (ifil, isng, irom, etc.)
                                Each sub-chunk: 4-byte ID + 4-byte size + data

  [INFO_end..INFO_end+3]        LIST chunk ID ("LIST")
  [+4..+7]                      LIST chunk size
  [+8..+11]                     List type ("sdta")
  [+12..]                       snam sub-chunk: 4-byte ID + 4-byte size + N*20 bytes
  [..]                          smpl sub-chunk: 4-byte ID + 4-byte size + PCM data
                                (smpl may be absent in ROM-only banks)

  [sdta_end..sdta_end+3]        LIST chunk ID ("LIST")
  [+4..+7]                      LIST chunk size
  [+8..+11]                     List type ("pdta")
  [+12..]                       Nine sub-chunks in order:
                                  phdr: N * 38 bytes
                                  pbag: N * 4 bytes
                                  pmod: N * 6 bytes (usually just 6 = terminal)
                                  pgen: N * 4 bytes
                                  inst: N * 22 bytes
                                  ibag: N * 4 bytes
                                  imod: N * 6 bytes (usually just 6 = terminal)
                                  igen: N * 4 bytes
                                  shdr: N * 16 bytes


8. SYNTHESIS HIERARCHY
-----------------------

The data structures form a three-level hierarchy:

  PRESET (phdr)
    |
    +-- Zone (pbag) -- contains generators (pgen) and modulators (pmod)
    |     |
    |     +-- Generator 41 (instrument) -> points to INSTRUMENT
    |     +-- Other generators: default values for the preset
    |
    +-- Zone (pbag) -- a preset may have multiple zones
          |
          +-- Generator 43 (keyRange) -> which MIDI keys use this zone
          +-- Generator 41 (instrument) -> points to another INSTRUMENT

  INSTRUMENT (inst)
    |
    +-- Zone (ibag) -- contains generators (igen) and modulators (imod)
    |     |
    |     +-- Generator 53 (sampleID) -> points to SAMPLE HEADER (shdr)
    |     +-- Generator 55 (sampleRate) -> playback rate in Hz
    |     +-- Generator 58 (overridingRootKey) -> root MIDI note
    |     +-- Generator 43 (keyRange) -> which keys trigger this sample
    |     +-- Generator 54 (sampleModes) -> loop mode
    |     +-- Other generators: envelope, filter, tuning parameters
    |
    +-- Zone (ibag) -- an instrument may have multiple zones (splits)

  SAMPLE (shdr + snam + smpl)
    |
    +-- Name from snam (20-byte record)
    +-- Boundaries from shdr (start, end, loopStart, loopEnd)
    +-- Audio data from smpl (16-bit signed LE PCM mono)

To find all parameters for a given note:
1. Find the preset matching the MIDI program number
2. Find the preset zone whose keyRange contains the note
3. Read the instrument index from generator 41
4. Find the instrument zone whose keyRange contains the note
5. Read generators from that zone (sample ID, rate, envelope, etc.)
6. Apply preset-level generators as defaults (overridden by instrument-level)


9. DIFFERENCES FROM SOUNDFONT 2.0
-----------------------------------

Feature                    SoundFont 1.0 (.SBK)       SoundFont 2.0 (.SF2)
-------                    --------------------       --------------------
File extension             .SBK                       .SF2
ifil version               1.0                        2.01 or 2.04
Sample header (shdr)       16 bytes (4 DWORDs)        46 bytes (20-char name +
                                                      start/end/loop/rate/pitch/
                                                      correction/link/type)
Sample names               Separate 'snam' chunk      Embedded in shdr records
                           (20 bytes per name)        (20 bytes per record)
Sample rate storage        Generator 55 in igen/pgen  dwSampleRate in shdr
Modulator record size      6 bytes                    10 bytes
Chunk alignment            No padding required        Word-aligned (2-byte)
Sample data format         16-bit signed LE mono      16-bit signed LE mono
                                                      (24-bit in SF2.04)
Target hardware            EMU8000 (AWE32/AWE64)      EMU10K1 (SB Live!) and
                                                      software synthesizers


10. NOTES
----------

- The EMU8000 chip requires inter-sample padding gaps in the smpl data
  (typically 40-76 silence frames between consecutive samples). These gaps
  are needed for the chip's 8-point sinc interpolation engine. The shdr
  start/end offsets skip these gaps.

- ROM-only banks (like the General MIDI bank shipped with AWE32 cards) have
  an snam chunk listing sample names but no smpl chunk. The shdr offsets
  refer to the EMU8000's 1MB or 2MB built-in ROM containing the GM sample
  set. These samples cannot be extracted from the SBK file.

- The pmod and imod chunks almost always contain only a single 6-byte
  terminal record (all zeros). Real-time modulation in the EMU8000 is
  primarily handled through its hardware registers rather than SBK file
  modulators.

- Generator values at the preset level serve as defaults that can be
  overridden by instrument-level generators. If a generator appears in
  both pgen and igen for the same note, the igen value takes precedence.

- The last record in phdr has the name "EOP" (End Of Presets). The last
  record in inst has the name "EOI" (End Of Instruments). These are
  terminal sentinels used to calculate the zone range of the preceding
  entry and must always be present.

- Some SBK creation tools (e.g., early Creative Labs utilities) may not
  initialize all fields in phdr or inst records, leaving garbage data in
  unused fields like dwLibrary, dwGenre, dwMorphology, or inst names.
  Parsers should not rely on these fields containing meaningful data.

- Sample rates in SF1 are typically in the range of 4000-13000 Hz, much
  lower than the 44100 Hz associated with CD-quality audio. This reflects
  the limited RAM available on AWE32 cards (typically 512KB-28MB). The
  EMU8000 chip pitch-shifts samples in real-time during playback.
