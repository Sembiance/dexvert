Notes:
- FormatID is the DrvInfo.ID value set by the format reader
- Offset is the byte position in the file where the magic check starts
- "n/a" means no magic bytes are checked (extension-only or structural matching)
- Some formats use checks beyond 16 bytes (DRSID=36 bytes, REZID=127 bytes, etc.) but are included if their first bytes start within 0-15
- The MIX format uses structural checks (field arithmetic) rather than fixed magic bytes
- FPK version varies (FPK1, FPK2, etc.) based on file header content
- BNT2 is detected by last 4 bytes of file, not first 16 - excluded
- SLF is detected at offset 520, not first 16 bytes - excluded
- Hub functions (ReadHubPAK, ReadHubWAD, etc.) re-check magic bytes for extension-based fallback
- EXPFORMATS (FORGE, NRM, H4R) are compile-time disabled and excluded
